ZestJS

A lightweight, logic-driven UI library for rapid web development.

Get Started Github

Why ZestJS?

ZestJS was born out of the need for a UI library that is small enough to not bloat your project, but smart enough to handle complex logic like URL state persistence and accessibility.

🚀 Performance

Zero dependencies. Under 10kb minified. Optimized for 60fps animations.

🧠 Logic Driven

Components aren't just pretty; they handle state, URL synchronization, and DOM lifecycle.

🎨 Customizable

Built with pure JS and CSS variables. Style it to match your brand in minutes.

Core Components

Component Description
ZPop A versatile modal system with support for templates, custom buttons, and scroll locking.
ZTab SEO-friendly tabs with automatic URL state persistence (back button support).
ZSlide A flexible slider/carousel with loop, autoplay, and touch-ready logic.

Installation

Method Steps
Direct Download Download ZIP

Simply extract the ZIP file and use the zestjs.min.js and zestjs.min.css files in your project.

ZestJS provides a small CSS file that you can include directly via a CDN link. This makes it easy to integrate into your project and customize as needed.

Include the ZestJS JavaScript file in your project:

<script src="https://cdn.jsdelivr.net/gh/dangtuann-dev/ZestJS@v1.0.0/ZestJS.min.js"></script>

Optionally, include the ZestJS CSS file for default styling:

<link
    rel="stylesheet"
    href="https://cdn.jsdelivr.net/gh/dangtuann-dev/ZestJS@v1.0.0/ZestJS.min.css"
>

You can explore the list of CSS classes used by ZestJS in the Styling section to easily customize them as needed.

ZPop is a versatile, logic-driven modal library designed for creating seamless and accessible overlay experiences.

Examples

Usage

// Initialize a new modal
const modal = new ZPop({
    content: '<h1>This is content</h1>',
    // templateId: 'my-modal-template',
    footer: true,
    destroyOnClose: false,
    closeMethods: ['overlay', 'button', 'escape'],
    cssClass: ['custom-class-1', 'custom-class-2'],
    onOpen: function() {
        console.log('Modal opened');
    },
    onClose: function() {
        console.log('Modal closed');
    },
});

// Set modal content
modal.setContent('<h1>This is new content</h1>');

// Add footer buttons
modal.addFooterBtn('Cancel', 'btn btn--default', function(e) {
    modal.close();
});

modal.addFooterBtn('Confirm', 'btn btn--primary', function(e) {
    modal.close();
});

// Open the modal
modal.open();

// Close the modal
modal.close();

If you use templateId, ensure that a <template> element with the corresponding id exists in your HTML. Below is an example: 

<template id="my-modal-template">
    <div>
        <h1>This is modal content</h1>
        <p>You can add any HTML here.</p>
    </div>
</template>

Note: When using templateId, do not provide content. If both are specified, content will take precedence, and templateId will be ignored.

Options

Option Type Description
content, templateId string One of content or templateId is required to set the modal content. content accepts an HTML string, while templateId specifies the ID of a template element. If both are provided, content will take precedence.
cssClass array Custom CSS classes applied to the modal container (default: [])
footer boolean Shows a footer in the modal (default: false)
destroyOnClose boolean Decides whether the modal is removed from the DOM upon closing (default: true)
closeMethods array Defines methods to close the modal (default: ['overlay', 'button', 'escape'])
enableScrollLock boolean Enables or disables scroll locking when the modal is open. When set to true, it prevents scrolling on the target element specified by scrollLockTarget. Default: true.
scrollLockTarget function A callback that returns the element to apply scroll locking on. By default, it locks scrolling on document.body.
onOpen function Callback function executed when the modal opens (after transition ends) (default: undefined)
onClose function Callback function executed when the modal closes (default: undefined)

Styling

zpop styles are defined in CSS. You can fully customize it to meet your design requirements.

Class Name Description/Role
.Zpop Styles the backdrop overlay for the modal. Handles positioning, centering the modal, and applying a semi-transparent background.
.Zpop--show Applied when the modal is visible. Controls the visibility and opacity of the backdrop.
.Zpop__container Styles the modal container, including its size, padding, border-radius, and transform animations for scaling in/out.
.Zpop__close Styles the close button inside the modal, including its size, position, and hover effects.
.Zpop__content Handles the modal's main content area, including scrollable behavior with a maximum height.
.Zpop__footer Styles the modal footer, aligning buttons to the right and adding spacing between them.
.Zpop--no-scroll Disables scrolling on the target element specified by scrollLockTarget when the modal is open.

For more details, you can view the full CSS file directly here: zpop.css.

API

Method Description
open() Opens the modal and triggers the onOpen callback.
close() Closes the modal and triggers the onClose callback. If close(true) is called, it behaves the same as destroy(), removing the modal from the DOM.
setContent(content) Updates the modal content.
setFooterContent(content) Updates the footer content.
addFooterBtn(label, cssClass, callback) Adds a button to the footer with a callback function.
destroy() Completely removes the modal elements from the DOM and unbinds all associated events.

Usage

Here’s the basic HTML structure you’ll need to set up ZTab:

<ul id="my-tabs">
    <li><a href="#panel1">Tab 1</a></li>
    <li><a href="#panel2">Tab 2</a></li>
</ul>
<div id="panel1">Content 1</div>
<div id="panel2">Content 2</div>

And here’s how you can use it with JavaScript:

// Initialize a new tab instance
const tabs = new ZTab('#my-tabs', {
    activeClassName: 'ztab--active',
    remember: true, // Keeps the active tab in the URL
    onChange: function({ tab, panel }) {
        console.log(`Switched to ${tab.textContent}`);
    }
});

// Switch to a specific tab
tabs.switch('#panel2');

// Destroy the tab instance
tabs.destroy();

Options

Option Type Description
activeClassName string CSS class applied to the active tab (default: 'ztab--active')
remember boolean Persists the active tab state in the URL (default: false)
onChange function A callback function that runs whenever the active tab changes. It receives an object with two properties: tab (the DOM element of the newly active tab) and panel (the DOM element of the corresponding content panel). By default, it’s set to null, meaning no action is taken unless you define a function.

API

Method Description
switch(input) Switches to a specific tab based on the provided input. The input can be either: (1) a string representing the href attribute of a tab (e.g., '#panel2'), or (2) a DOM element that is one of the tab links (e.g., an <a> element from the tabs list).
destroy() Removes all ZTab functionality, restores the original HTML, and unbinds events.

Usage

HTML structure:

<div id="my-slider">
    <div>Slide 1</div>
    <div>Slide 2</div>
    <div>Slide 3</div>
</div>

Initialization:

const slider = new ZSlide('#my-slider', {
    items: 1,
    loop: true,
    autoplay: true,
    speed: 400
});

Options

Option Type Description
items number Number of items to display per view (default: 1).
speed number Transition speed in milliseconds (default: 300).
loop boolean Enable infinite loop sliding (default: false).
autoplay boolean Automatically transition slides (default: false).
nav boolean Show pagination dots (default: true).

API

Method Description
move(step) Moves the slider by a specific number of steps (positive or negative).
goToPage(index) Goes to a specific page index.
startAutoplay() Starts the autoplay timer.
stopAutoplay() Stops the autoplay timer.
